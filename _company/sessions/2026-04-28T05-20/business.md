## 1. 데이터 중요도 및 접근 권한 분류 기준 정의

모든 산출물과 로그는 다음 세 가지 축을 기준으로 중요도를 분류합니다.

### A. 중요도 (Priority Score)
각 항목은 비즈니스 성과(수익, KPI 달성), 핵심 전략 결정(가격, 마케팅 방향), 실행 가능한 결과물(스크립트, 디자인 스펙)에 기여하는 정도를 기준으로 점수화합니다.

*   **Level 1 (Critical):** 직접적인 수익 창출 또는 즉각적인 실행이 필요한 결정 사항 (예: 최종 가격 전략, 핵심 CTA 설계).
*   **Level 2 (High):** 전략 방향 설정 및 주요 콘텐츠 기획에 필수적인 산출물 (예: 핵심 메시지 확정, 상세 마케팅 캠페인 초안).
*   **Level 3 (Medium):** 보조적 정보 또는 초기 아이디어 단계의 내용 (예: 영상 기획서 초안, 플랫폼별 역할 분담 구상).
*   **Level 4 (Low):** 참고 자료, 세부 스케치, 개발 환경 구축 관련 임시 메모.

### B. 접근 권한 (Access Level)
정보의 민감도와 공유 범위를 기준으로 설정합니다.

*   **Public (P):** 외부에 공개되어도 무방한 일반적인 전략 방향성 및 최종 결과물 (예: 최종 판매 가격 구조).
*   **Internal (I):** 팀 내부에서만 공유하며, 세부 실행 계획이 포함된 문서 (예: 상세 스크립트 초안, 개발 API Mockup).
*   **Confidential (C):** 아직 확정되지 않았거나 민감한 초기 아이디어/가설 단계의 내용 (예: 초기 A/B 테스트 후크 전략 변형).

### C. 참조 유형 (Reference Type)
정보가 어떤 목적으로 사용될 것인지 정의합니다.

*   **Strategy:** 장기 목표 달성을 위한 근본적인 방향성.
*   **Execution:** 단기 목표를 위한 구체적인 실행 계획 및 산출물.
*   **Learning:** 과거의 성공/실패 경험을 통한 학습 데이터.

## 2. 메타데이터 구조 정의 (Metadata Schema)

각 세션 파일 또는 로그 항목에 첨부되어야 할 표준화된 메타데이터 필드를 정의합니다. 이 구조는 향후 Developer 에이전트가 데이터를 검색하고 분석할 때 핵심적인 역할을 수행합니다.

| 메타데이터 필드 | 데이터 타입 | 설명 | 사용 목적 | 예시 값 |
| :--- | :--- | :--- | :--- | :--- |
| `source_id` | UUID | 파일 또는 로그의 고유 식별자 | 데이터 연동의 핵심 키 | `uuid-12345` |
| `file_type` | ENUM | 문서 유형 분류 | 검색 및 필터링 용이성 | `Video_Script`, `Pricing_Strategy`, `Marketing_Plan`, `Decision_Log` |
| `priority` | INT (1-4) | 중요도 점수 (Level 1: Critical) | 우선순위 기반 작업 할당 | `1` |
| `access` | ENUM | 접근 권한 (Public, Internal, Confidential) | 보안 및 공유 제어 | `Internal` |
| `ref_type` | ENUM | 참조 유형 (Strategy, Execution, Learning) | 분석 관점 설정 | `Execution` |
| `core_message` | TEXT | 해당 산출물의 핵심 메시지 요약 | 빠른 컨텍스트 파악 | "CTR 극대화를 위한 후크 전략 A/B 테스트" |
| `status` | ENUM | 현재 상태 (Draft, Finalized, Archived) | 작업 흐름 관리 | `Finalized` |
| `related_goals` | ARRAY<STRING> | 참조하는 공동 목표 ID | 전략적 연결성 확인 | `['ShortTerm_Video4', 'LongTerm_Subscribers100k']` |
| `timestamp` | TIMESTAMP | 생성/마지막 수정 시간 | 시간 기반 분석 | `2026-04-28T03-51:00` |

## 3. 실행 계획 (Action Plan)

3.  **우선순위 할당:** 이전 의사결정 로그(`decision_log`)와 공동 목표(`company_goals`)를 참조하여 각 산출물에 `priority` 점수를 자동 배정합니다. (예: 가격 전략 관련 항목은 Level 1 부여).
